10 common html mistakes made by beginners

As a beginner in web development, it is often very exciting writing html elements that make things happen just as you want it to happen without much stress But there are times it can become very frustrating, when you try out a punch of tags, elements and nothing happens the way you want it to be.Especially when you know you have done everything right.
Now, let's jump into what some of the mistakes, one makes especially as a newbie.

1. Forgetting the DOCTYPE

DOCTYPE shows browsers the version of HTML you are using. In the absence of DOCTYPE, the browser makes assumptions on the kind of code used.
Just imagine a toodler, seeing white paint in a container and thinking it is yogurt 😬 not good, right??? Unless you label the container as paint, the toodler will make assumptions.
That is the same scenario when a browser comes in contact with a document without a DOCTYPE. We all can agree that the results of such a document won't be as good as we expected.
So, to avoid making such mistakes consider the points below;
  • Use DOCTYPE, irrespective of the kind of HTML you use in your code
  • Always place DOCTYPE in the first line of your source code
  • Take note, DOCTYPE is case-sensitive

2. Neglecting Character Encoding

Character sets simply tell the web browser the kind of characters used in the page. The character encoding tells the browser what kind of data to read and display. Let's take an example; A page containing the English characters found on our keyboards will have a different character set from the one that displays Japanese or Chinise characters. This is where character encoding comes into play. It’s important to specify a character set of the document as earlier as possible otherwise the browser will be left to determine the encoding before parsing the document and this may negatively affect the page load time. For most English Web pages, the character encoding will be entered into the Web page like this: <meta charset="utf-8">.This meta tag should be within the <head> and </head> tags of your Web page and is not case sensitive.

4. Wrong placement of Block Elements

Before we get into this, it will be of great importance we understand that, each HTML element is either displayed in Block or Inline.
Block elements are elements that always start on a new line and takes up the full width availableon the web page, effectively blocking out any other element from being close to it
Inline elements are elements that don't start in a new line and only take up width as necessary, thereby allowing other elements to be in line with the inline element.

5. Weak CSS structuring

Most at times, many newbie developers focus on content and website visual elements instead of focusing on organising CSS styles in a logical manner. Beginners usually write CSS in the order they think, as a result a style declaration can't be found when it is time to do changes.
💡 Always organise your CSS code by structuring the CSS according to the organisation of HTML you are working on. For example;
  • header,
  • body,
  • article,
  • section,
  • sidebar,
  • and footer

6. Not using 'alt' attribute in the img tag

Including the alt attribute in an image (img) tag is of vital importance. You should be wondering why the highlight on it's importance. Okay, let me tell you why. Before we get to that, let's refresh our minds on what the role of 'alt' attribute actually is. The ALT attribute enables you as the developer to provide alternative information for an image in case a user's device can't load the image properly maybe due to poor internet connection or an error in the 'src' attribute or if the user uses a screen reader

  • The primary purpose of the ALT text is to describe images to visistors who are unable to see them. These include, screen readers and browsers that block images.
  • The content of ALT attribute helps Search engines to understand what an image is all about

7. Confusing use of Classes and IDs

Classes are for items that are used more than once on one page. This can be a link style that you’ll call in multiple times on one page but doesn’t follow the global link styling.

Ids are items that are called just once, like the header div.

8. Not using unique names for Ids and classes

Choosing names that are unique is of great importance when it comes to Ids and Classes. Using unique names makes it easy to edit later and also easy to identify in the stylesheet. For example, naming a div as #home-right-column is better than just #right

9. Incorrect use of positioning

When first starting out with CSS, positioning is tough to understand. There are five types of positioning values:static,relative,fixed, absolute and sticky
